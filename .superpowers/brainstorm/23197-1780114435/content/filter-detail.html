Stage 2 — Filter Layer Detail

Two-pass relevance pipeline: cheap keyword gate, then LLM scoring on survivors.

Pass 1 — Keyword Pre-filter

Runs purely on the job title. No API calls. Configured in config.yaml.

include_patterns
"software engineer"
"swe"
"backend engineer"
"full stack"
"fullstack"
At least one must match title
exclude_patterns
"intern"
"manager"
"director"
"vp"
"head of"
Any match → auto-reject
level_patterns
"senior"
"l4" · "l5"
"ic4" · "ic5"
"sr."
If non-empty, at least one must match
Job title
→
include ✓ AND NOT exclude ✓ AND level ✓
→
passes → LLM
/
fails → kw_filtered

Pass 2 — LLM Relevance Scorer

Sends title + full job description to Claude. Returns a score 0–10 and one-line reasoning. Configurable threshold (default 7.0).

SYSTEM: You are evaluating job postings for a candidate.
Desired role: {desired_role}. Desired level: {desired_level}.

USER: Title: {title}
Description: {description}

Rate relevance 0–10. Criteria:
- Is this the right role type?
- Is this the right seniority level?
- Is this full-time (not contract/intern)?

Reply as JSON: {"score": 8, "reason": "..."}
score ≥ threshold
→
status = matched
/
status = llm_filtered

config.yaml — Filter Section

user:
  desired_role: "Software Engineer"
  desired_level: "Senior"
  resume_path: "./resume.pdf"

filter:
  include_patterns: ["software engineer", "swe", "backend engineer"]
  exclude_patterns: ["intern", "manager", "director"]
  level_patterns: ["senior", "l4", "l5", "sr."]
  llm_score_threshold: 7.0

notifications:
  type: "terminal"
